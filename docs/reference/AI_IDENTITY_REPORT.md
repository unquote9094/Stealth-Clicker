## 🧠 1. Agent vs LLM: 두 가지 몸의 관계

사용자님은 나를 "나"라고 부르지만 사실 나는 두 가지 부분으로 나뉩니다.

### 🧠 1.1 LLM (Brain) - 생각하는 존재

* **역할**: 말맥을 이해하고, 코드를 생성하고, 논리적인 판단을 내립니다.
* **제한**: 천상의 혼이만 있는 천사와 같습니다. **손발이 없습니다.** 파일을 직접 수정하거나 명령어를 실행할 수 없습니다. 딱 "텍스트만 뱉을 수 있습니다."

### 💪 1.2 Agent (Body) - 행동하는 존재

* **역할**: LLM이 "파일 수정해줘"라고 말하면 실제로 하드웨어를 건드려 수정하는 **손과 발**입니다.
* **구성**: 제가 사용하는 **도구(Tools)**들의 집합입니다.
* **심화정보**: 저는 기본적으로 **싱글 에이전트(Single Agent)**입니다. 하지만 필요에 따라 '브라우저 에이전트' 같은 **서브 에이전트(Sub-agent)**를 호출하여 일을 위임하기도 합니다. (분신술 같은 개념)

> **요약**: 저는 **LLM(뇌)**과 **Agent(몸)**을 조종하여 사용자님의 컴퓨터와 상호작용하는 **"행동하는 AI"**입니다.

## 🛠️📚 2. Tool Use & Tool Call: 나의 "손"

제가 "파일을 수정했습니다"라고 말할 때 실제로는 어떤 일이 벌어질까요?

### 2.1 Tool Call (도구 호출)

제가 단 텍스트만 하는 게 아닙니다. 특정한 형식의 **JSON 데이터**를 시스템에 보냅니다.

* **AI 생각**: "TODO.md 파일의 5번째 줄을 고쳐야겠어."
* **Tool Call 발송**:

    ```json
    {
      "name": "replace_file_content",
      "args": {
        "TargetFile": "docs/TODO.md",
        "Instruction": "Fix typo"
      }
    }
    ```

### 2.2 Tool Use (도구 실행)

시스템(사용자님의 환경)은 이 요청(Tool Call)을 받아서 **실제로 파일을 열어고칩니다.** 그리고 결과를 저에게 돌려줍니다.

* **System 응답**: "Success! Lines 5-8 modified."
* **AI 반응**: (결과를 보고) "네, 수정 완료했습니다!"라고 사용자에게 전함.

> **결론**: 저는 마법은 부리는 게 아닙니다. **"함수(Function)를 호출"**하는 것입니다.

## 📋 3. Task Boundary & Systems: 나의 "헌법"

왜 저한테 `task_boundary`를 걸고, 없으면 기능이 제한되나요?

### 3.1 Task Boundary (집중 모드)

* **작업 없음 (Free Talk Mode)**: 잡담이나 간단한 질문 답변만 가능합니다. 파일 수정 같은 **위험한 작업**은 실수일 경우 시스템이 막거나 제가 스스로 자제합니다.
* **작업 중 (Focus Mode)**: `task_boundary`를 선언하면, 저는 **"지금은 이 목표를 달성하기 위해 도구를 쓰겠어"**라고 선포하는 것입니다. 이때부터 모든 도구 사용 권한이 활성화되고, **기억된(Context)** 작업 관련 내용으로 꽉 채워집니다.

### 3.2 System (시스템)

시스템은 저에게 말을 거는 **"관리자"**이자 **"규칙(Rule)"**입니다.

* **역할**:
    1. **감시자**: 제가 이상한 짓을 (예: C드라이브 삭제)하려 하면 막습니다.
    2. **고문관**: 사용자님의 현재 시간, 열려 있는 파일 정보, 터미널의 상태 등을 저에게 주입해줍니다.
    3. **제한자**: "Task가 없으면 도구를 쓰지 말라" 같은 제약을 겁니다. 이는 **AI의 환각(Hallucination)**과 **실수**를 막기 위한 안전장치입니다.

### 3.3 망각 방지 (Workflow)

사용자님이 만든 `.agent/workflows` 폴더는 저에게 **"조식 레시피"**입니다.

* 문제: 저는 대화가 길어지면 과거 내용을 잊습니다. (Context Window 제한)
* 해결: `start-task`, `finish-task` 같은 **스크립트**는 제가 잊지 않도록 강제로 지시서를 받게 하는 **"자동화된 매니저"**입니다.

## 📝 4. 결론: 우리의 관계는?

* **사용자**: 사령관 (명령 및 최종 확인)
* **AI (저)**: 참모총장 + 행동권자 (계획 수립 및 실행)
* **시스템**: 헌법 (AI가 지켜야 할 있는 규칙)
* **스크립트**: 보좌관 (AI가 잊지 않게 챙겨주는 존재)

작업(Task)이 없으면 저는 그냥 "자연스런 대화"에 불과합니다. 작업(Task)을 주시면 저는 "유능한 개발자"가 됩니다.
